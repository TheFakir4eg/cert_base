Cert Base

Веб-приложение для управления движением сертификатов (купоны на скидку) внутри организации: эмиссия, выдача, использование(списание), вывод из оборота.
Проект демонстрирует полный цикл разработки backend-сервиса: от проектирования БД до деплоя и администрирования.


В компаниях часто существует процесс выдачи различных разрешений/допусков к блоку информации, который ведётся в Excel, почте и мессенджерах.
Цель проекта — перевести этот процесс в централизованную систему с ролями, аудитом и удобным интерфейсом.

Основной функционал
- авторизация и управление пользователями
- роли и группы пользователей
- система разрешений (permissions)
- выдача и учёт сертификатов
- админ-панель для управления справочниками
- seed-скрипты для наполнения базы
- логирование и миграции БД
- docker-окружение для запуска

Стек технологий

Backend:

- Python
- Flask
- SQLAlchemy
- Flask-Migrate (Alembic)
- MariaDB

Frontend:

- Jinja2 templates
- Bootstrap
- Vanilla JS

Инфраструктура:

- Docker / Docker Compose
- Gunicorn
- Nginx (планируется/используется в прод-сценарии)
- VPS-деплой
  
Архитектура

Проект построен по классической структуре Flask-приложения:

app/
- models — ORM модели
- routes — HTTP endpoints
- services — бизнес-логика
- templates / static — фронтенд
- migrations — миграции базы данных
- docker-окружение для запуска всего стека

Разделение на слои:

- routes → принимают запросы
- services → бизнес-логика
- models → работа с БД
  
Запуск проекта

- git clone https://github.com/TheFakir4eg/cert_base.git
- cd cert_base
- docker compose up --build

После запуска:

- приложение поднимает БД
- выполняются миграции
- можно запустить seed-скрипты для тестовых данных

Этот репозиторий показывает мой практический опыт:

разработка production-подобного Flask-сервиса
проектирование БД и миграции
реализация RBAC/permissions
работа с Docker и деплоем на VPS
поддержка и развитие существующего проекта
